# xyd-
方向一垃圾分类识别系统
基于 ResNeXt50 的生活垃圾多分类对比实验项目完整介绍
一、项目背景与研究目的
1. 项目背景
垃圾分类是城市环保、资源回收再利用的核心环节，传统人工分拣存在效率低、人力成本高、分拣标准不统一等问题。借助计算机视觉 + 深度学习实现垃圾自动图像分类，可落地智能垃圾桶、流水线分拣设备，降低人工依赖。
本项目为人工智能工程实训项目，基于公开 TrashNet 垃圾分类数据集，使用卷积神经网络完成 6 类生活垃圾识别，并通过三组超参数对照实验，分析学习率、优化器、批次大小对模型精度、训练速度、显存占用的影响，掌握深度学习调参、训练监控、性能对比完整流程。
2. 核心目标
搭建 ResNeXt50 迁移学习分类模型，完成纸板、玻璃、金属、纸张、塑料、其他垃圾 6 分类任务；
设计 3 组对照实验，对比不同学习率、优化器、BatchSize 的训练效果；
使用 TensorBoard 全程监控 Loss、准确率、单轮耗时、显存峰值，可视化训练曲线；
输出量化对比表格，总结最优超参数组合，分析过拟合、收敛速度、硬件开销规律。

三、模型架构：ResNeXt50_32x4d
ResNeXt系列网络结构
核心原理：ResNeXt 是 ResNet 升级变体，采用分组卷积（Cardinality 基数 32），在不显著增加参数量的前提下，提取更丰富的图像纹理、轮廓特征，适合中小规模垃圾数据集；
迁移学习方案：加载 ImageNet 预训练权重（本项目权重置空从零训练，也可开启预训练提升精度），替换最后一层全连接层适配 6 分类任务；
损失函数：交叉熵损失CrossEntropyLoss，适配多分类任务；
学习率调度：余弦退火调度器CosineAnnealingLR，动态降低学习率，提升后期收敛精度。
四、三组对照实验设计（项目核心创新点）
固定模型为 ResNeXt50，仅调整学习率、优化器、BatchSize三个变量，形成三组控制变量实验，每组完整训练 30 轮：
表格
实验组名称	学习率 lr	优化器	BatchSize	实验目的
exp1_lr1e-4_Adam_b16	1e-4	Adam	16	基准对照组，常用小学习率 Adam 配置
exp2_lr1e-5_Adam_b16	1e-5	Adam	16	对比极低学习率对收敛速度、最终精度影响
exp3_lr1e-4_SGD_b32	1e-4	SGD	32	对比优化器、更大批次对显存、准确率的影响
五、项目完整功能模块（你的全部代码功能）
1. 训练循环模块
GPU 自动加速，无 GPU 自动切换 CPU 运行；
每轮训练统计训练损失、训练准确率；验证集不开启梯度，统计验证损失、验证准确率；
硬件监控：每轮重置 CUDA 显存统计，记录单轮训练耗时、GPU 显存峰值；
最优模型保存：每轮验证精度提升时，自动保存xxx_best.pth权重文件。
2. TensorBoard 日志监控模块（你重点使用的功能）
每一轮训练完成后，通过add_scalar写入 6 类指标日志，存入runs/实验名文件夹：
Loss/train：训练损失曲线
Loss/val：验证损失曲线
Acc/train：训练集准确率
Acc/val：验证集准确率
Hardware/epoch_time：每轮训练耗时
Hardware/max_memory_GB：每轮显存占用峰值
配套writer.flush()强制写入磁盘，避免日志丢失；训练结束writer.close()完整关闭日志流；
启动命令读取总runs文件夹，网页同时加载 3 组实验曲线，直观对比收敛差异。
3. 实验结果汇总模块
全部三组实验跑完后，自动打印格式化汇总表格，包含：实验名称、学习率、优化器、批次大小、最佳验证准确率、平均单轮耗时、平均显存峰值，直观量化三组模型性能与硬件开销。

七、预期实验结论
学习率影响：exp1（1e-4）收敛速度更快、最终精度更高；exp2（1e-5）学习率过小，收敛缓慢，验证准确率偏低；
优化器对比：Adam 自适应学习率收敛更快；SGD 搭配大批次（32）显存占用更高，稳定后精度接近 Adam；
硬件开销：BatchSize=32 的 exp3 显存峰值显著高于 16 批次的两组；Adam 单轮训练速度略快于 SGD；
最优组合：exp1（lr=1e-4，Adam，batch=16）综合精度、训练速度、显存开销最优，为本项目推荐超参数配置。
八、项目难点与现存问题
日志空白问题：训练中途手动终止程序，flush/close未执行，tfevents 文件仅空壳无标量数据，TensorBoard 无法渲染曲线（你当前遇到的核心问题）；
数据集类别不均衡：“其他垃圾” 样本量远少于其余 5 类，模型容易偏向大类，小类识别精度偏低；
显存限制：BatchSize 调大时 GPU 内存溢出，训练进程静默中断，日志写入逻辑中断；
相对路径坑：代码初始使用相对路径生成日志，不同电脑 PyCharm 工作目录不一致，导致 TensorBoard 找不到日志文件（其他电脑能出图、本机空白的原因）。
